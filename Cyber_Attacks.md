# Cyber Attacks

## About

A Cyber Attack is an attempt by an individual or group to gain unauthorized access to a computer system, network, or data with the intention of stealing information, disrupting services, or causing damage.

Cyber attacks can target individuals, businesses, governments, and critical infrastructure.

---

# Objectives of Cyber Attacks

- Steal Sensitive Data
- Financial Gain
- Disrupt Business Operations
- Damage Reputation
- Spy on Organizations
- Gain Unauthorized Access

---

# Common Types of Cyber Attacks

## 1. Phishing

### Definition

Phishing is a social engineering attack in which attackers trick users into revealing sensitive information through fake emails, websites, or messages.

### Example

An email pretending to be from a bank asks the user to verify login credentials through a fake website.

### Prevention

- Verify sender addresses
- Do not click suspicious links
- Enable Multi-Factor Authentication (MFA)
- Use email security filters

---

## 2. Denial of Service (DoS)

### Definition

A DoS attack floods a system with excessive requests from a single source, making it unavailable to legitimate users.

### Prevention

- Firewalls
- Rate Limiting
- Traffic Filtering

---

## 3. Distributed Denial of Service (DDoS)

### Definition

A DDoS attack uses multiple compromised devices (botnets) to overwhelm a target server or network.

### Prevention

- DDoS Protection Services
- Load Balancing
- Web Application Firewalls (WAF)
- Traffic Monitoring

---

## 4. Man-in-the-Middle (MitM)

### Definition

In a MitM attack, an attacker secretly intercepts communication between two parties.

### Example

Intercepting traffic on an unsecured public Wi-Fi network.

### Prevention

- HTTPS
- VPN
- Strong Encryption
- Avoid Public Wi-Fi for sensitive activities

---

## 5. Brute Force Attack

### Definition

Attackers repeatedly try different password combinations until the correct password is found.

### Prevention

- Strong Passwords
- Account Lockout Policies
- MFA
- CAPTCHA

---

## 6. Password Attack

Common password attacks include:

- Dictionary Attack
- Credential Stuffing
- Password Spraying
- Keylogging

### Prevention

- Unique Passwords
- Password Managers
- MFA
- Regular Password Updates

---

## 7. SQL Injection (SQLi)

### Definition

Attackers insert malicious SQL queries into application input fields to manipulate or access databases.

### Example

Bypassing authentication or retrieving confidential records.

### Prevention

- Parameterized Queries
- Input Validation
- Least Privilege Database Accounts

---

## 8. Cross-Site Scripting (XSS)

### Definition

XSS allows attackers to inject malicious scripts into web pages viewed by other users.

### Types

- Stored XSS
- Reflected XSS
- DOM-Based XSS

### Prevention

- Input Validation
- Output Encoding
- Content Security Policy (CSP)

---

## 9. Zero-Day Attack

### Definition

A Zero-Day Attack exploits a software vulnerability before the vendor releases a security patch.

### Prevention

- Regular Updates
- Threat Monitoring
- Endpoint Protection
- Patch Management

---

## 10. Botnet Attack

### Definition

A Botnet is a group of compromised devices controlled remotely by an attacker.

### Uses

- DDoS Attacks
- Spam Campaigns
- Cryptocurrency Mining
- Malware Distribution

---

## 11. Ransomware Attack

### Definition

Ransomware encrypts files or systems and demands payment to restore access.

### Prevention

- Regular Backups
- Updated Antivirus
- Security Awareness Training
- Patch Management

---

## 12. Malware Attack

Types of Malware

- Virus
- Worm
- Trojan
- Spyware
- Adware
- Rootkit
- Ransomware

      ↓

      ↓

      ↓
Privilege Escalation
      ↓
Lateral Movement
      ↓
Data Exfiltration
      ↓

---

# Indicators of a Cyber Attack

- Slow System Performance
- Unknown Programs Running
- High Network Traffic
- Unauthorized Login Attempts
- Unexpected File Changes
- Disabled Security Software
- Suspicious Emails

---

# Prevention Best Practices

- Keep Software Updated
- Use Strong Passwords
- Enable MFA
- Install Antivirus
- Use Firewalls
- Backup Important Data
- Encrypt Sensitive Information
- Educate Users About Phishing
- Monitor Network Activity

---

# Real-Life Example

A user receives an email claiming to be from a bank. The email contains a fake login page.

The user enters their username and password.

The attacker steals the credentials and accesses the bank account.

This is a **Phishing Attack**.
