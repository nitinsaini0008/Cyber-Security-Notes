# CIA Triad

## About

The CIA Triad is the foundation of Information Security and Cyber Security. It consists of three core security principles:

- Confidentiality
- Integrity
- Availability

Every security policy, control, and technology is designed to protect one or more of these principles.

---

# What is the CIA Triad?

The CIA Triad is a security model used to protect information and information systems from unauthorized access, modification, or disruption.

```
        CIA TRIAD

     Confidentiality
            ▲
           / \
          /   \
         /     \
Integrity ----- Availability
```

---

# 1. Confidentiality

## Definition

Confidentiality ensures that information is accessible only to authorized users.

Unauthorized users must not be able to view sensitive information.

---

## Examples

- Online Banking
- Medical Records
- Company Payroll
- Government Documents

---

## Methods to Maintain Confidentiality

- Password Protection
- Multi-Factor Authentication (MFA)
- Encryption
- Access Control
- File Permissions
- VPN

---

## Threats to Confidentiality

- Data Breach
- Phishing
- Insider Threats
- Password Theft
- Packet Sniffing

---

# 2. Integrity

## Definition

Integrity ensures that information remains accurate, complete, and unaltered.

Only authorized users should be able to modify data.

---

## Examples

- Bank Account Balance
- Student Exam Results
- Company Financial Records

---

## Methods to Maintain Integrity

- Hashing
- Digital Signatures
- Checksums
- File Permissions
- Version Control
- Audit Logs

---

## Threats to Integrity

- Malware
- SQL Injection
- Unauthorized Changes
- Data Corruption
- Insider Attacks

---

# 3. Availability

## Definition

Availability ensures that systems and data are accessible whenever authorized users need them.

---

## Examples

- Hospital Systems
- ATM Networks
- Cloud Services
- E-commerce Websites

---

## Methods to Maintain Availability

- Data Backup
- Disaster Recovery
- Redundant Servers
- Load Balancing
- UPS (Uninterruptible Power Supply)
- Regular Maintenance

---

## Threats to Availability

- DDoS Attacks
- Hardware Failure
- Power Failure
- Ransomware
- Natural Disasters

---

# CIA Triad Summary

| Principle | Goal | Examples |
|------------|------|----------|
| Confidentiality | Prevent Unauthorized Access | Encryption, Passwords |
| Integrity | Prevent Unauthorized Modification | Hashing, Digital Signatures |
| Availability | Ensure Continuous Access | Backup, Redundancy |

# Real-Life Example

Imagine an online banking system.

### Confidentiality

Only the account holder can view account details.

### Integrity

No one can change the account balance without authorization.

### Availability

Customers should be able to access online banking 24×7.
